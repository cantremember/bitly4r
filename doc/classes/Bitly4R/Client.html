Client

A client object for accessing the bit.ly API.

  • Supports both API key and HTTP Auth credentials (although HTTP Auth / password-based doesn‘t seem to work)
  • Works with version 2.0.1 of the API
  • Uses XML for marshalling, for cheap & easy text parsing.

See the API documentation:

  • code.google.com/p/bitly-api/wiki/ApiDocumentation
  • bit.ly

expand(param, param_type=:short_url)

Invokes the API‘s expand method. It reverses a shorten; the original full URL is re-hydrated.

For param, you can provide a previously-shortened URL from the bit.ly service. If so, you do not need to provide param_type (though :short_url would be the proper symbol).

Alternately, you can provide a hash code returned by the bit.ly service. In this case, provide :hash as the param_type.

A Response is returned. Response.to_s will return the longUrl / long_url value.

info(param, param_type=:short_url, ops={})

Invokes the API‘s info method. Information about the shortened URL is returned by the service.

For param, you can provide a previously-shortened URL from the bit.ly service. If so, you do not need to provide param_type (though :short_url would be the proper symbol).

Alternately, you can provide a hash code returned by the bit.ly service. In this case, provide :hash as the param_type.

You can optionally provide an arbitrary Hash of HTTP parameters. The one that bit.ly cares about is called :key. It should be an array of camel-cased or underscored element names which you would like to receive. In theory, this should limit the response content, but I haven‘t seen that work yet. The arbitrary Hash capability is left intact for future re-purposing.

A Response is returned. Response.to_s will return the longUrl / long_url value.

There is plenty of other data in the response besides the original full URL. Feel free to access the Response.body and use Response.method_missing to pull out specific element values.

shorten(long_url)

Invokes the API‘s shorten method. That‘s pretty much what makes bit.ly a valuable service.

A Response is returned. Response.to_s will return the shortUrl / short_url value.

You can take the shortened URL and re- expand it.
