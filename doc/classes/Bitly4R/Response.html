Response

A response from the bit.ly API.

The to_s method should always return the ‘likely’ value, assuming that there is a likely one. For example:

  • shorten => shortUrl
  • expand => longUrl

All other response values can be retrieved via method_missing.

NOTE: This is not a sophisticated XML parser. It‘s just Regexp‘s, with handling for CDATA blocks.

Public Class methods

new(response, to_s_sym=nil)

Constructs a bit.ly API response wrapper.

response can be:

  • a String, which becomes the body
  • a Net::HTTPResponse, in which case its body is extracted

to_s_sym is optional, and it references the element which will become the to_s value of this Response. It can be either camel-case or underscored. See method_missing.

Provides access the other text elements of the response via camel-case or underscored names. For example, longUrl and long_url are equivalent. If no such element exists, you‘ll get nil.
